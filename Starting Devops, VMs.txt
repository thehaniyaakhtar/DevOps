Localhost
Localhost refers to the computer you are currently working on. It uses a loopback address (typically `127.0.0.1`) that allows the machine to 
communicate with itself over a network interface. It can also be accessed using the domain name `localhost`.

Ping and Domain Resolution
The `ping` command is used to check connectivity between your system and a domain. It also resolves the domain name to its corresponding IP address, 
helping verify whether the target server is reachable.

IP Address (IPv4)
An IPv4 address consists of four octets separated by dots, where each octet ranges from 0 to 255 (e.g., `192.168.1.1`). 
It uniquely identifies a device on a network.

Local Network and Private IPs
Devices connected to the same router (e.g., multiple laptops on the same Wi-Fi network) can communicate with each other using private IP addresses. 
This enables local hosting and resource sharing within the network without exposing services to the internet. (mild hosting)

Network Configuration (`ifconfig`)
The `ifconfig` command displays network interfaces and their configurations:
`lo0`: The loopback interface (localhost communication).
`en0`: Typically the Wi-Fi interface, showing the device’s local/private IP address.

Serving Files on a Port
Running a local server binds your file system to a specific port (e.g., `localhost:3000`). Other devices on the same network can access it using 
your private IP address and port. However, open ports can be discovered through port scanning, which may expose hosted files if not secured.

DNS Manipulation and Phishing Risk
When you open a website, your system resolves a domain name to an IP address. If this mapping is altered locally (e.g., via a hosts file), 
a legitimate domain can be redirected to a fake server. This technique can be used to mimic real websites and potentially steal user credentials.

Self-Hosting vs Cloud Hosting
Self-hosting involves running services on your own hardware, giving full control but requiring maintenance and setup. 
Cloud hosting provides managed infrastructure, scalability, and convenience, but comes at a higher cost.

Virtual Machines (VMs)
A virtual machine is a software-based emulation of a physical computer. A hypervisor sits on top of physical hardware (CPU, RAM, storage) and 
allocates resources dynamically to multiple VMs. Each VM operates independently with its own operating system.

Bare Metal Servers
A bare metal server is a physical machine dedicated to a single user or organization. It does not use a hypervisor, which allows maximum performance 
and control. However, resources cannot be dynamically shared or reallocated as in virtualized environments. 
This setup is often used in high-performance workloads such as blockchain mining.

SSH (Secure Shell) Protocol
SSH is a cryptographic network protocol used to securely connect to remote systems. It allows users to log into remote machines, 
execute commands, and transfer files securely.

Key features:
Encryption: Protects data during transmission.
Authentication: Supports password-based and public key-based authentication.
Integrity: Ensures data is not altered during transmission.

SSH Key-Based Authentication
In public key authentication:
* A key pair (public and private key) is generated.
* The public key is stored on the remote machine in the `~/.ssh/authorized_keys` file.
* The private key remains on the client machine.

When connecting using `ssh user@ip_address`, the server verifies the client using the stored public key, allowing secure access without a password.
