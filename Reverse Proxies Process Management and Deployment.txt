Elastic Computing
Elastic computing refers to the ability to dynamically allocate and release computing resources (CPU, RAM, storage) based on demand.
This is a core feature of cloud platforms like Amazon Web Services, allowing systems to scale up or down without manual hardware changes.

It also includes:
Auto scaling (automatically adding/removing instances based on load)
Load balancing (distributing traffic across multiple instances)
EC2 Instance Basics

An EC2 (Elastic Compute Cloud) instance is a virtual machine in the cloud.

Each instance:
Runs inside a Virtual Private Cloud (VPC)
Has:
Private IP (for internal communication)
Public IP (for internet access, if enabled)
Instance States

An EC2 instance can exist in different states:
Pending: Instance is being launched
Running: Instance is active and usable
Stopped: Instance is shut down but not deleted (can be restarted)
Terminated: Instance is permanently deleted (cannot be recovered)
Reboot: Restart without losing data
Elastic IP

An Elastic IP is a static public IP address that can be attached to an EC2 instance.
Unlike regular public IPs, it does not change when the instance is restarted, making it useful for stable access.

Additional points:
Must be manually attached
Limited in number
Used for stable domain mapping
Security Groups

Security Groups
Security groups act as virtual firewalls for your EC2 instance.
They control inbound and outbound traffic.

You define rules based on:
Protocol (e.g., TCP)
Port (e.g., 22 for SSH, 80 for HTTP, 3000 for apps)
Source (IP ranges allowed)

Example:

Allow TCP on port 22 → enables SSH access
Allow TCP on port 3000 → enables access to your app

Important:
Security groups are stateful
Default:
All inbound blocked
All outbound allowed

Best practice:
Avoid 0.0.0.0/0 for SSH (restrict to your IP)
Instance Types

Instance types define the hardware configuration of your VM (CPU, memory, networking capacity).

Categories:
General purpose
Compute optimized
Memory optimized
Storage optimized
Steps to Create an EC2 Instance
Go to EC2 dashboard → Launch Instance
Assign a name to the VM
Select an Operating System (OS) and version (e.g., Ubuntu)
Choose an instance type (based on required resources)
Create or select a key pair (used for SSH authentication)
Configure security groups:
Open port 22 (SSH)
Open port 3000 (or any app port you need)
Add storage (e.g., 15 GB)
Launch the instance

You then connect using:
ssh -i filename.pem ubuntu@public_ip

Notes:
Username depends on OS (ubuntu, ec2-user, etc.)
Key permissions must be restricted
File Permissions (Linux)

Each file has permissions that control access.

Example:
-rw-r--r--

This represents:
Owner: read (r), write (w)
Group: read (r)
Others: read (r)
Permission Values (RWE System)
Read (r) = 4
Write (w) = 2
Execute (x) = 1

Examples:
7 (111) → read + write + execute
6 (110) → read + write
4 (100) → read only

Common cases:
644 → normal files
755 → executable files
Fixing Key Permission Error

Private key files must be restricted:
chmod 400 filename.pem

(600 also works; prevents others from reading the key)

Nginx (Web Server & Reverse Proxy)

Nginx is an open-source software used for:
Web serving
Reverse proxying
Load balancing
Caching
SSL termination (handling HTTPS)
Ports and Web Traffic
HTTP runs on port 80
HTTPS runs on port 443

When a user visits a website without specifying a port, the browser automatically uses these defaults.

If your app runs on port 3000, users must visit:
http://your-ip:3000
Reverse Proxy Concept

Only one service can directly listen on a specific port (e.g., 80).

To handle multiple applications:

Run apps on different ports:
App A → port 3000
App B → port 3001

Use Nginx as a reverse proxy:
Nginx runs on port 80
It forwards requests based on URL

Example:
/app1 → forwarded to port 3000
/app2 → forwarded to port 3001

Example config:

location /app1 {
    proxy_pass http://localhost:3000;
}

location /app2 {
    proxy_pass http://localhost:3001;
}

This allows multiple services to appear under one domain.

Production Insight (Important)

After adding Nginx:
Close port 3000 in security group
Only expose:
80 (HTTP)
443 (HTTPS)

This prevents direct access to backend services.

Installing and Running Nginx
sudo apt update
sudo apt install nginx
sudo systemctl start nginx
sudo systemctl enable nginx

Reload after config changes:
sudo systemctl reload nginx
End-to-End Flow
User hits domain → resolves to Elastic IP
Request reaches EC2 instance
Nginx listens on port 80/443
Nginx forwards request to backend (3000/3001)
Backend processes request
Response returns via Nginx
